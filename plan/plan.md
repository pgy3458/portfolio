## 1. 프로젝트 목표 요약

사용자가 자연어로 투자 내역을 입력하면 AI가 이를 해석하고, 확인 절차를 거쳐 Google Sheets에 데이터를 저장하며, 사이트 내 모든 탭(포트폴리오 현황·거래 이력·AI 분석)에서 직접 조회할 수 있는 웹 기반 자산 관리 시스템.

**핵심 가치:**
- 자연어 입력 → 정형 데이터 변환 (입문자도 쉽게 사용)
- 확인(Confirm) 절차를 통한 데이터 정확성 보장
- Apple 감성의 미니멀하고 직관적인 UX
- 투자 이론(행동경제학, 리스크 관리)이 내재된 구조

## 2. 기술 스택 결정

| 영역 | 선택 기술 | 선택 이유 |
|------|----------|----------|
| **Frontend** | HTML + CSS + Vanilla JS (단일 파일) | 초보자 배려, 복잡도 최소화, 빠른 구현 |
| **AI / NLP** | Gemini API (또는 GPT-4o) | 한국어 자연어 이해 성능, 비용 효율 |
| **배포 환경** | GitHub Pages (정적 호스팅) | 무료, 24시간, 모바일·맥북 동시 접근 |
| **인증** | 비밀번호 입력 (내부 해시 비교) | 사용자는 비밀번호만 입력, 내부 처리는 비노출 |
| **중간 레이어** | Google Apps Script | API 키 노출 없이 Sheets 읽기·쓰기, 무료 |
| **스프레드시트 연동** | Google Sheets API (Apps Script 경유) | 데이터 저장소 + 포트폴리오 자동 계산 유지 |

## 3. 시스템 아키텍처

```
[사용자 웹 브라우저 — 맥북 / 모바일]
       │  비밀번호 입력 (최초 화면)
       │  인증 성공 시 전체 탭 표시
       ▼
[Frontend — GitHub Pages (index.html)]
  ├─ 탭 1: 입력 (자연어 거래 입력)
  ├─ 탭 2: 포트폴리오 현황
  ├─ 탭 3: 거래 이력
  ├─ 탭 4: AI 분석
  └─ 탭 5: 오류
       │
       ├─ AI API 직접 호출 (Gemini)
       │
       └─ HTTP POST → Google Apps Script (중간 레이어)
                    │
                    ▼
            [Google Sheets — 데이터 저장소]
              ├─ Sheet 1: 거래 이력 + 거시경제
              ├─ Sheet 2: 포트폴리오 현황 (수식 자동 계산)
              └─ Sheet 3: AI 분석 결과
```

## 4. 개발 단계 (Phase)

### Phase 1 — 환경 설정 및 기반 구축
**목표**: 개발 환경 세팅 및 Google Sheets 연결 확인

- [ ] GitHub 저장소 생성 및 GitHub Pages 활성화
- [ ] Google Apps Script 프로젝트 생성 및 웹 앱 배포
- [ ] 비밀번호 인증 로직 설정 (해시값 config.js 내 저장)
- [ ] Google Sheets API 활성화
- [ ] `.env` 파일로 API 키 및 스프레드시트 ID 관리 (보안)
- [ ] 테스트용 구글 스프레드시트 생성 및 시트 3개 구조 초기화

**산출물**: Google Apps Script ↔ Google Sheets 연결 테스트

### Phase 2 — 스프레드시트 구조 설계 및 초기화
**목표**: 3개 시트의 컬럼 구조 확정 및 초기 데이터 포맷 설정

#### Sheet 1: Raw Sheets (거래 및 거시경제 이력)
| 컬럼 | 내용 |
|------|------|
| 날짜 | 거래 발생일 |
| 종목명 | 예: 삼성전자, Apple, Toyota |
| 국가/시장 | KR / US / JP |
| 자산유형 | 주식 / ETF / 현금 |
| 액션 | 매수 / 매도 |
| 수량 | 거래 수량 |
| 단가 | 거래 단가 (원화 기준) |
| 총금액 | 수량 × 단가 |
| 누적 보유수량 | 해당 종목 누적 |
| 평균단가 | 자동 계산 |
| USD/KRW 환율 | 거래 시점 |
| USD/JPY 환율 | 거래 시점 |
| 한국금리(기준) | 거래 시점 |
| 미국금리(기준금리) | 거래 시점 |
| 일본금리 | 거래 시점 |
| KOSPI | 거래 시점 |
| NASDAQ | 거래 시점 |
| Nikkei | 거래 시점 |
| 메모 | 사용자 입력 원문 |

#### Sheet 2: Dashboard Main (포트폴리오 현황)
| 섹션 | 내용 |
|------|------|
| 글로벌 마켓 지표 | 실시간 환율 3종, 금리 3개국, 주요 지수 3개 |
| 총 자산 현황 | 투자원금 / 현재가치 / 총수익률 / 현금비중 |
| 보유 종목 테이블 | 종목별 평균단가·현재가·평가손익·비중 |
| 자산 배분 현황 | 위험자산 vs 안전자산 비중 (색상 구분) |

#### Sheet 3: AI Analysis (AI 분석 및 경고)
| 섹션 | 내용 |
|------|------|
| 리스크 지표 | MDD(최대낙폭), 변동성(표준편차), 샤프지수 |
| 종목 간 상관계수 | 분산 효과 측정 |
| 자동 경고 | 특정 종목 비중 초과 / 손실 임계값 도달 |
| 리밸런싱 제안 | 목표 비중 대비 현재 비중 이탈 시 알림 |
| 행동경제학 경고 | 손실회피 패턴, 특정 종목 집중 편향 감지 |
| AI 코멘트 | 입력 시점 거시경제 기반 간략 분석 |

### Phase 3 — 사이트 디자인 + UI 제작
**목표**: Apple 스타일 미니멀 웹 인터페이스 구현 (더미 데이터로 화면 구성)

**디자인 원칙:**
- 불필요한 선 제거, 카드 레이아웃, 샌드박스(San Francisco) 계열 서체
- 상단 진행 바: Step 1(입력) → Step 2(AI 해석 중) → Step 3(확인) → Step 4(완료)
- 버튼 색상: 단계별 상태에 따라 유기적으로 변화 (회색 → 파란색 → 초록색)
- 금융 데이터 계층화: 핵심 수치(굵고 진함) vs 보조 수치(얇고 연함)
- 반응형 디자인: 데스크탑(맥북) + 모바일(아이폰) 동시 지원

**화면 구성:**
1. 최초 접속 화면: 비밀번호 입력창만 표시 (인증 전 탭 비노출)
2. 로딩 화면: AI 해석 중 애니메이션
3. 확인 화면: 파싱 결과 카드 (수정 가능) + 거시경제 데이터 미리보기
4. 완료 화면: 성공 피드백 + 포트폴리오 요약 링크
5. 오류 탭: 오류 발생 위치와 내용을 명시하는 전용 섹션
6. 포트폴리오 탭: 보유 종목 현황·수익률 표시
7. 거래 이력 탭: 전체 거래 내역 조회

### Phase 4 — NLP 파서 개발
**목표**: 자연어 입력 → 정형 데이터 변환 모듈

- [ ] AI API (Gemini 또는 GPT) 연동 설정
- [ ] 프롬프트 엔지니어링: 한국어/영어 혼용 투자 언어 처리
- [ ] 파싱 결과 형식 정의 (JSON 스키마)
- [ ] 오류 처리: 필수값 누락 시 사용자에게 재입력 요청
- [ ] 날짜 처리: "오늘", "어제", "3일 전" 등 상대적 표현 변환

**파싱 예시:**
```
입력: "오늘 삼성전자 10주 8만원에 매수했어"
출력: {
  "date": "2026-03-28",
  "stock_name": "삼성전자",
  "ticker": "005930",
  "market": "KR",
  "action": "매수",
  "quantity": 10,
  "price": 80000,
  "total": 800000
}
```

### Phase 5 — 거시경제 데이터 수집 모듈 개발
**목표**: 거래 시점의 거시경제 데이터 자동 수집

- [ ] 환율 수집: `exchangerate-api` 또는 한국은행 API (USD/KRW, USD/JPY)
- [ ] 금리 수집: FRED API (미국), 한국은행 API (한국), 일본은행 API (일본)
- [ ] 지수 수집: `yfinance` 라이브러리 (KOSPI: ^KS11, NASDAQ: ^IXIC, Nikkei: ^N225)
- [ ] 수집 실패 시 빈칸 처리 (시스템 중단 없이 계속 진행)

### Phase 6 — 연결 (Apps Script — 시트 ↔ 사이트)

- [ ] Apps Script 엔드포인트로 거래 데이터 전송 구현
- [ ] 응답 결과 확인 후 UI 피드백 처리
- [ ] Dashboard Main 자동 갱신 (수식 기반)
- [ ] AI Analysis 트리거 (새 거래 후 리스크 지표 재계산)

## 5. 파일 구조

```
city-making/
│
├── index.html
│   └─ 브라우저에서 열리는 페이지
│      탭 전체(입력·포트폴리오·거래이력·AI분석·오류) 포함
│      최초 접속 시 비밀번호 입력창만 표시
│      인증 성공 시 전체 탭 노출
│
├── style.css
│   └─ 디자인 전담 파일
│      색상, 폰트, 레이아웃, Apple UX 스타일 적용
│
├── app.js
│   └─ 기능 전담 파일
│      비밀번호 인증 처리 (해시 비교)
│      탭 전환 제어
│      Gemini AI API 호출 (자연어 파싱)
│      Apps Script 통신 (Sheets 읽기·쓰기)
│      거시경제 데이터 수집
│
├── config.js
│   └─ 설정값 전담 파일
│      비밀번호 해시값 저장
│      Apps Script 웹 앱 주소 저장
│      (이 파일만 수정하면 설정 변경 완료)
│
├── _ai/
│   │  ※ AI 전용 폴더 — 브라우저에서 열리지 않음
│   │
│   ├── CONTEXT.md
│   │   └─ 새 Claude Code 세션 시작 시 가장 먼저 읽는 파일
│   │      프로젝트 현재 상태, 기술 스택, 다음 할 일 요약
│   │      (매 세션마다 이 파일 하나로 프로젝트 재설명 불필요)
│   │
│   └── CHANGELOG.md
│       └─ 코드 변경 이력 기록
│          언제, 어떤 파일을, 왜 수정했는지 추적
│
└── _docs/
    │  ※ 문서 전용 폴더 — 브라우저에서 열리지 않음
    │
    ├── plan.md
    │   └─ 본 계획서 (현재 파일)
    │
    ├── REVIEW_LOG.md
    │   └─ 코드 작성 전 검토·승인 기록
    │      변경 요청 내용과 승인 여부 보관
    │
    └── TODO.md
        └─ 우선순위별 남은 작업 목록
           완료 체크 및 다음 작업 관리
```

**Google Sheets (구글 드라이브 내 별도 존재 — 로컬 폴더 아님)**
```
포트폴리오 스프레드시트
  ├── Sheet 1: 거래 이력 + 거시경제  ← app.js가 Apps Script 통해 읽기·쓰기
  ├── Sheet 2: 포트폴리오 현황       ← 수식 자동 계산, 사이트 탭 2에서 표시
  └── Sheet 3: AI 분석 결과         ← 사이트 탭 4에서 표시
```
