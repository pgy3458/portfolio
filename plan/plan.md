# City Making — 자산 관리 자동화 시스템 개발 계획서

> **상태**: 승인 대기 중 (코드 작성 전)
> **작성일**: 2026-03-28
> **기반 문서**: research.md

---

## 1. 프로젝트 목표 요약

사용자가 자연어로 투자 내역을 입력하면 AI가 이를 해석하고, 확인 절차를 거쳐 구글 스프레드시트의 포트폴리오를 자동으로 업데이트하는 웹 기반 자산 관리 시스템.

**핵심 가치:**
- 자연어 입력 → 정형 데이터 변환 (입문자도 쉽게 사용)
- 확인(Confirm) 절차를 통한 데이터 정확성 보장
- Apple 감성의 미니멀하고 직관적인 UX
- 투자 이론(행동경제학, 리스크 관리)이 내재된 구조

---

## 2. 기술 스택 결정

| 영역 | 선택 기술 | 선택 이유 |
|------|----------|----------|
| **Frontend** | HTML + CSS + Vanilla JS (단일 파일) | 초보자 배려, 복잡도 최소화, 빠른 구현 |
| **Backend** | Python (Flask) | 가벼운 서버, 초보자 친화적, gspread 연동 용이 |
| **AI / NLP** | Gemini API (또는 GPT-4o) | 한국어 자연어 이해 성능, 비용 효율 |
| **스프레드시트 연동** | gspread + Google Sheets API v4 | research.md 명시 기술 |
| **배포 환경** | 로컬(localhost) → 추후 Render 또는 Railway 배포 | MacBook Air M5 로컬 우선 |
| **인증** | Google Service Account (JSON 키) | 서버-투-서버 인증, 사용자 로그인 불필요 |

## 3. 시스템 아키텍처

```
[사용자 웹 브라우저]
       │
       │ 자연어 입력 (예: "오늘 삼성전자 10주 8만원에 매수했어")
       ▼
[Frontend — index.html]
  ├─ 입력창 (단일 검색창 형태)
  ├─ 진행 상태 표시 바 (Step 1→2→3)
  └─ 확인 화면 (파싱 결과 카드 미리보기)
       │
       │ HTTP POST /parse
       ▼
[Backend — Flask 서버 (app.py)]
  ├─ NLP 모듈 (nlp_parser.py)
  │    └─ AI API 호출 → {날짜, 종목, 수량, 가격, 액션} 추출
  ├─ 거시경제 수집 모듈 (macro_fetcher.py)
  │    └─ 환율(USD/KRW, USD/JPY), 금리, 지수(KOSPI, NASDAQ, Nikkei) 수집
  └─ 스프레드시트 모듈 (sheets_writer.py)
       └─ gspread → Google Sheets API 호출
       │
       ▼
[Google Sheets — 포트폴리오]
  ├─ Sheet 1: Raw Sheets (거래 이력 + 거시경제 이력)
  ├─ Sheet 2: Dashboard Main (현황 요약 + 보유 종목)
  └─ Sheet 3: AI Analysis (리스크 지표 + 경고 + 리밸런싱)
```

### Phase 2 — 스프레드시트 구조 설계 및 초기화
**목표**: 3개 시트의 컬럼 구조 확정 및 초기 데이터 포맷 설정

#### Sheet 1: Raw Sheets (거래 및 거시경제 이력)
| 컬럼 | 내용 |
|------|------|
| 날짜 | 거래 발생일 |
| 종목명 | 예: 삼성전자, Apple, Toyota |
| 국가/시장 | KR / US / JP |
| 자산유형 | 주식 / ETF / 현금 |
| 액션 | 매수 / 매도 |
| 수량 | 거래 수량 |
| 단가 | 거래 단가 (원화 기준) |
| 총금액 | 수량 × 단가 |
| 누적 보유수량 | 해당 종목 누적 |
| 평균단가 | 자동 계산 |
| USD/KRW 환율 | 거래 시점 |
| USD/JPY 환율 | 거래 시점 |
| 한국금리(기준) | 거래 시점 |
| 미국금리(기준금리) | 거래 시점 |
| 일본금리 | 거래 시점 |
| KOSPI | 거래 시점 |
| NASDAQ | 거래 시점 |
| Nikkei | 거래 시점 |
| 메모 | 사용자 입력 원문 |

#### Sheet 2: Dashboard Main (포트폴리오 현황)
| 섹션 | 내용 |
|------|------|
| 글로벌 마켓 지표 | 실시간 환율 3종, 금리 3개국, 주요 지수 3개 |
| 총 자산 현황 | 투자원금 / 현재가치 / 총수익률 / 현금비중 |
| 보유 종목 테이블 | 종목별 평균단가·현재가·평가손익·비중 |
| 자산 배분 현황 | 위험자산 vs 안전자산 비중 (색상 구분) |

#### Sheet 3: AI Analysis (AI 분석 및 경고)
| 섹션 | 내용 |
|------|------|
| 리스크 지표 | MDD(최대낙폭), 변동성(표준편차), 샤프지수 |
| 종목 간 상관계수 | 분산 효과 측정 |
| 자동 경고 | 특정 종목 비중 초과 / 손실 임계값 도달 |
| 리밸런싱 제안 | 목표 비중 대비 현재 비중 이탈 시 알림 |
| 행동경제학 경고 | 손실회피 패턴, 특정 종목 집중 편향 감지 |
| AI 코멘트 | 입력 시점 거시경제 기반 간략 분석 |

### Phase 3 — NLP 파서 개발
**목표**: 자연어 입력 → 정형 데이터 변환 모듈

- [ ] AI API (Gemini 또는 GPT) 연동 설정
- [ ] 프롬프트 엔지니어링: 한국어/영어 혼용 투자 언어 처리
- [ ] 파싱 결과 형식 정의 (JSON 스키마)
- [ ] 오류 처리: 필수값 누락 시 사용자에게 재입력 요청
- [ ] 날짜 처리: "오늘", "어제", "3일 전" 등 상대적 표현 변환

**파싱 예시:**
```
입력: "오늘 삼성전자 10주 8만원에 매수했어"
출력: {
  "date": "2026-03-28",
  "stock_name": "삼성전자",
  "ticker": "005930",
  "market": "KR",
  "action": "매수",
  "quantity": 10,
  "price": 80000,
  "total": 800000
}
```

### Phase 4 — 거시경제 데이터 수집 모듈 개발
**목표**: 거래 시점의 거시경제 데이터 자동 수집

- [ ] 환율 수집: `exchangerate-api` 또는 한국은행 API (USD/KRW, USD/JPY)
- [ ] 금리 수집: FRED API (미국), 한국은행 API (한국), 일본은행 API (일본)
- [ ] 지수 수집: `yfinance` 라이브러리 (KOSPI: ^KS11, NASDAQ: ^IXIC, Nikkei: ^N225)
- [ ] 수집 실패 시 빈칸 처리 (시스템 중단 없이 계속 진행)

### Phase 5 — 확인(Confirm) 플로우 및 스프레드시트 쓰기 모듈
**목표**: 파싱 결과를 사용자에게 확인받은 후 최종 기록

- [ ] 파싱 결과 → 확인 카드 UI 생성
- [ ] 사용자 수정 가능 필드 제공 (수량, 가격 등 직접 수정 허용)
- [ ] "확인 후 저장" 버튼 클릭 시에만 스프레드시트 업데이트
- [ ] Raw Sheets에 거래 내역 + 거시경제 데이터 한 행으로 기록
- [ ] Dashboard Main 자동 갱신 (수식 기반)
- [ ] AI Analysis 트리거 (새 거래 후 리스크 지표 재계산)

---

### Phase 6 — Frontend UI 개발
**목표**: Apple 스타일 미니멀 웹 인터페이스 구현

**디자인 원칙:**
- 불필요한 선 제거, 카드 레이아웃, 샌드박스(San Francisco) 계열 서체
- 상단 진행 바: Step 1(입력) → Step 2(AI 해석 중) → Step 3(확인) → Step 4(완료)
- 버튼 색상: 단계별 상태에 따라 유기적으로 변화 (회색 → 파란색 → 초록색)
- 금융 데이터 계층화: 핵심 수치(굵고 진함) vs 보조 수치(얇고 연함)
- 반응형 디자인: 데스크탑(맥북) + 모바일(아이폰) 동시 지원

**화면 구성:**
1. 메인 입력 화면: 단일 검색창 + 간단한 입력 예시
2. 로딩 화면: AI 해석 중 애니메이션
3. 확인 화면: 파싱 결과 카드 (수정 가능) + 거시경제 데이터 미리보기
4. 완료 화면: 성공 피드백 + 포트폴리오 요약 링크
5. 오류 탭: 오류 발생 위치와 내용을 명시하는 전용 섹션

---

## 5. 파일 구조 (모듈화)

```

├── app.py                  # Flask 메인 서버 (라우팅)

├── requirements.txt        # 필요 라이브러리 목록
│
├── modules/
│   ├── nlp_parser.py       # 자연어 → 정형 데이터 변환
│   ├── macro_fetcher.py    # 거시경제 데이터 수집
│   ├── sheets_writer.py    # 구글 스프레드시트 쓰기
│   └── error_logger.py     # 오류 기록 및 분류
│
├── static/
│   ├── style.css           # 전체 스타일 (Apple UX)
│   └── app.js              # 프론트엔드 로직
│
└── templates/
    └── index.html          # 메인 웹 페이지
```
