# 인증 흐름 설계
> 연관: 탭 0 · config.js

## 인증 흐름
1. 접속 → 탭 0만 표시 (탭바 비노출)
2. 비밀번호 입력 → SHA-256 해시 변환 (브라우저 내 처리)
3. config.js 저장값과 비교
4. 일치 → sessionStorage 저장 → 탭 1~5 표시
5. 불일치 → 오류 메시지 → 탭 비노출 유지

## 상태별 탭 노출
| 상태 | 표시 |
|------|------|
| 미인증 | 탭 0만 |
| 인증 성공 | 탭 1~5 전체 |
| 인증 실패 | 탭 0 + 오류 메시지 |

## 세션 유지
- sessionStorage 사용 (탭·브라우저 닫으면 초기화)
- 새 탭 열어도 재인증 필요
- 자동 로그인 없음
- 비밀번호 평문 코드 내 미포함 (해시값만 저장)
